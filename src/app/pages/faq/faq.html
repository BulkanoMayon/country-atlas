Frequently Asked Questions

Learn more about how the Country Atlas platform works, including its features, limitations, and how data is presented within the system.

What kind of system is this project?

It is a front-end travel and lifestyle exploration application built using Angular. It combines country data, destination highlights, map access, and user features.

Does the project use a backend?

No. This system is fully front-end only. All data is either retrieved from public APIs or stored locally using browser storage technologies.

How does login and registration work?

User accounts are simulated using localStorage. This means data is stored only in the browser and is not connected to any external server or database.

What happens if I clear my browser data?

All stored information including user accounts and favorites will be removed since the system relies on browser storage.

What kind of travel information is available?

Each country includes attractions, cities, food, cultural elements, events, and suggested activities, making it a lifestyle-oriented travel guide.

Can I view locations on a map?

Yes. Each country details page includes a direct link to map services so you can explore the destination visually.

Can I save my favorite countries?

Yes. You can add and remove countries from your favorites list. This data is stored locally and tied to your browser session.

Is this project complete?

The system demonstrates core features such as routing, API integration, UI design, and simulated user interaction. Additional improvements can still be added in future versions.
